## ‚ú¶ Overview

**Vault Bot** is an enterprise-level, fully async Telegram file storage system built with `python-telegram-bot v21+` and MongoDB. It combines the power of Google Drive, a search engine, an encrypted personal vault, a CDN-style file sharing service, and a SaaS admin dashboard ‚ÄĒ all inside Telegram.

Every interaction is keyboard-driven, every UI element uses **small-caps Unicode**, and every feature is production-ready with proper error handling, logging, and modular architecture.

**Design Principles:**
- **Separation of Concerns** ‚ÄĒ handlers never touch the DB directly; all logic lives in services
- **Async everywhere** ‚ÄĒ `motor`, `asyncio`, `python-telegram-bot` v21 concurrent updates
- **Typed** ‚ÄĒ dataclasses, type hints, and enums throughout
- **Zero external file storage** ‚ÄĒ all files stored as Telegram `file_id` in a private channel

## ‚ú¶ Quick Setup

### 1 ¬∑ Prerequisites

```bash
Python 3.11+
MongoDB Atlas (free tier works) or self-hosted MongoDB 7+
A Telegram Bot Token from @BotFather
A private Telegram channel for storage
A private Telegram channel for logs
```

### 2 ¬∑ Installation

```bash
git clone https://github.com/youruser/telegram-vault-bot
cd telegram-vault-bot

python -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

### 3 ¬∑ Environment

```bash
cp .env.example .env
nano .env                      # fill in all values
```

Key variables:

| Variable | Description |
|---|---|
| `BOT_TOKEN` | From @BotFather |
| `OWNER_ID` | Your Telegram user ID |
| `ADMIN_IDS` | Comma-separated admin IDs |
| `MONGO_URI` | MongoDB connection string |
| `STORAGE_CHANNEL_ID` | Private channel ID (with `-100` prefix) |
| `LOG_CHANNEL_ID` | Private log channel ID |
| `REQUIRED_CHANNELS` | e.g. `secretsbotz` (no @) |
| `UPI_ID` | Your GPay/UPI ID e.g. `yourname@gpay` |

### 4 ¬∑ Storage Channel Setup

1. Create a private Telegram channel
2. Add your bot as **admin** with post permissions
3. Copy the channel ID (format: `-100XXXXXXXXXX`)
4. Set it as `STORAGE_CHANNEL_ID` in `.env`

### 5 ¬∑ Run

```bash
python main.py
```



## ‚ú¶ Security

- **PIN hashing** ‚ÄĒ vault PINs are hashed with SHA-256 + secret salt
- **Fernet encryption** ‚ÄĒ vault metadata is symmetrically encrypted
- **Rate limiting** ‚ÄĒ token bucket (10 messages/60s default, configurable)
- **Role-based access** ‚ÄĒ owner > admin > premium > user > banned
- **Input sanitisation** ‚ÄĒ all user inputs sanitised before DB operations
- **Session timeout** ‚ÄĒ vault auto-locks after 30 minutes of inactivity
- **Owner bypass** ‚ÄĒ owner ID bypasses all restrictions
- **Channel enforcement** ‚ÄĒ bot unusable until required channels are joined
